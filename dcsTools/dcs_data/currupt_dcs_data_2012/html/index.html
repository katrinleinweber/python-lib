DcsData Reports

About the reports (click to open/close)

The structure of the reports reflects the repository structure:

  • repository
  • format
  • collection
  • record

    • The report shows only the corrupt records, and only branches that contain corrupt records. E.g., if there are no corrupt dates in a particular format, then that format is not shown in the report. likewise at the collection level.

    Each heading in the report (representing repository, format, and collection) includes a number in parens. This number represents the total number of corrupt record descendants "within" this heading.

    In the reports below, the collections and records are hyperlinked.

    • collection links do a search for corrupt records in that collection, and take you to the NCS to view the records as search results.

    • record links take you to the NCS to view the record as a search result.
